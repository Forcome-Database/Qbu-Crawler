# D019 报告生产语义与趋势修复

日期：2026-04-26

## 背景

生产测试 4 暴露出日报报告在 bootstrap / baseline 阶段的展示语义不清：累计评论容易被理解为本次新增，Excel 评论明细旧列名继续使用“本次新增”，全景热力图和趋势价格辅图有数据但未展示，变化趋势的“周/月/年”和维度命名也容易让用户误解为自然周期或每日监控趋势。

## 实现

- 基线期继续保持 `report_semantics = bootstrap`，新增 `baseline_day_index` / `baseline_display_state` / `window_meaning` 作为展示上下文。
- 总览 KPI 将自有评论改为累计口径，并补充累计竞品评论、本次入库评论、近30天评论。
- Excel 评论明细将“本次新增”改为“窗口归属”，基于 `window_review_ids` 区分本次入库与历史累计，并叠加新近/补采口径。
- 全景数据恢复 `charts.heatmap` 的 HTML table 渲染。
- 趋势页渲染 `secondary_charts`，允许主图仍在积累时展示已 ready 的价格辅图。
- 变化趋势视角显示为近7天、近30天、近12个月；维度显示为评论声量与情绪、问题结构、产品状态、竞品对标。
- 产品状态趋势基于 `scraped_at`，表格增加当前价格、当前库存、价格变化次数、库存变化次数、最近库存变化，并优先选择有价格/库存状态变化的重点 SKU。
- Excel 趋势数据保留产品快照明细，同时按 `trend_digest` 输出业务分块。

## 验证

- P0 / P1 / P2 相关定向 pytest 已通过。
- 生产报告代码和模板已增加门禁，禁止继续暴露旧列名“本次新增”。
