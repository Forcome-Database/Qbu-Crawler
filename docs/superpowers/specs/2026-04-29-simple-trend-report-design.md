# 历史口径变化趋势页设计

## 目标

变化趋势页用于让用户一眼看出：整体口碑是否变好，问题是否集中，哪些产品正在变差，自有产品和竞品的差距是否扩大。

页面保留测试5版本的“时间 + 维度”切换框架，但减少指标和图表数量。每个面板只回答一个问题，不做复杂的多层解释。

## 核心口径

日报中两类信息必须分开：

- 今日变化：只看本次 run / 本次入库 / 当天窗口，由 `change_digest` 提供。
- 变化趋势：只看历史库，截止到本报告 `data_until`，由 `trend_digest` 提供。

变化趋势不能只依赖本次新采集评论。即使当天没有新增评论，只要历史库有足够数据，趋势页也应能展示历史趋势。

## 数据来源

评论类趋势使用历史评论：

- 表：`reviews` + `products` + 最新 `review_analysis`
- 时间轴：优先 `reviews.date_published_parsed`，缺失时再用 `reviews.date_published`
- 可用性截止：只允许使用 `reviews.scraped_at < report.data_until` 的已入库评论
- 趋势分桶截止：只允许使用发布时间早于 `report.data_until` 的评论
- 发布时间为日期粒度时，按该日期 00:00:00 与 `report.data_until` 比较，不能只比较 `date(report.data_until)`，避免非零点截止时间误删当天数据。
- 用途：口碑趋势、问题趋势、竞品对比

产品类趋势使用历史产品状态：

- 表：`product_snapshots` + `products`
- 时间轴：`product_snapshots.scraped_at`
- 截止：只允许使用 `scraped_at < report.data_until` 的数据
- 产品身份优先使用唯一 `product_url` / `product_id`，不能只用 SKU 聚合；多站点或竞品场景下 SKU 可能重复。
- 用途：产品评分、评论总数、价格、库存等连续采集状态

当前 `products` 表只表示最新状态，不能单独作为历史趋势来源。

## 展示结构

顶部两排切换：

- 时间：近7天、近30天、近12个月
- 维度：口碑趋势、问题趋势、产品趋势、竞品对比

每个维度固定三块内容：

- 一张主图：展示趋势
- 最多三个关键指标：展示当前窗口状态和核心变化
- 一张明细表：解释是谁导致趋势变化

默认打开：近30天 / 口碑趋势。

## 通用文案规则

用户报告中不展示技术词和模糊词：

- 不展示“快照”，页面中使用“历史记录”或“连续采集数据不足”
- 不展示“声浪 / 声量”，统一使用“评论量”或“新增评论数”
- 不展示“上期”，统一使用明确时间窗口
- 新工作台和旧 fallback 展示都必须遵守禁词规则；不能只在 `workspace` 存在时满足。

对比窗口：

- 近7天：当前7天 vs 前7天，文案为“较前7天”
- 近30天：当前30天 vs 前30天，文案为“较前30天”
- 近12个月：当前12个月 vs 前12个月，文案为“较前12个月”

历史不足时显示“暂无足够历史对比”，不展示伪精确变化百分比。

## 口碑趋势

用途：判断整体口碑走势和自有产品是否落后竞品。

主图：

- 自有平均评分 vs 竞品平均评分

关键指标：

- 自有平均评分
- 自有差评率
- 与竞品评分差

明细表：

- 日期 / 月份
- 评论量
- 自有均分
- 自有差评率
- 竞品均分

计算：

- 自有平均评分 = 当前窗口自有评论评分均值
- 竞品平均评分 = 当前窗口竞品评论评分均值
- 自有差评率 = 自有产品 <=2 星评论数 / 自有产品评论数
- 与竞品评分差 = 竞品平均评分 - 自有平均评分

## 问题趋势

用途：判断问题是否集中、是否升级、影响哪些产品。

主图：

- Top 3 问题评论占比趋势

关键指标：

- 问题评论占比
- Top 1 问题
- 影响产品数

明细表：

- 问题
- 当前评论数
- 占比
- 对比变化
- 影响 SKU

计算：

- 只统计自有产品负面标签
- 问题评论占比 = 该问题评论数 / 当前窗口自有评论数
- Top 1 问题 = 当前窗口评论数最高的问题
- 影响产品数 = 当前窗口内出现该负面问题的 SKU 数

## 产品趋势

用途：定位正在变差的 SKU。

主图：

- 重点风险产品评分趋势

关键指标：

- 评分下降产品数
- 差评率上升产品数
- 评论增长但评分下降产品数

明细表：

- 产品
- 当前评分
- 评分变化
- 当前差评率
- 差评率变化
- 评论增长数

计算：

- 评分变化 = 当前窗口末值 - 当前窗口初值
- 评论增长数 = 当前窗口末值评论总数 - 当前窗口初值评论总数
- 差评率来自历史评论按产品聚合，不能只靠 `product_snapshots`
- 评分下降产品数 = 评分变化 < 0 的自有产品数
- 差评率上升产品数 = 差评率变化 > 0 的自有产品数
- 评论增长但评分下降产品数 = 评论数增加且评分下降的自有产品数

## 竞品对比

用途：判断自有产品和竞品差距是否扩大，以及竞品哪里值得借鉴。

主图：

- 评分差趋势：竞品均分 - 自有均分

关键指标：

- 当前评分差
- 差评率差
- 竞品优势主题 Top 1

明细表：

- 竞品优势主题
- 竞品好评数
- 自有相关问题
- 启示

计算：

- 当前评分差 = 竞品均分 - 自有均分
- 差评率差 = 自有差评率 - 竞品差评率
- 竞品优势主题 Top 1 = 竞品正向标签最高频主题
- 自有相关问题只匹配同 label code 或同 taxonomy category；没有可靠匹配时显示“暂无匹配”
- 启示使用确定性文案，不调用 LLM

## 数据不足状态

任何维度都必须优先展示可解释状态：

- 无数据：暂无趋势数据
- 样本不足：趋势数据正在积累
- 历史不足：暂无足够历史对比

数据不足时可以展示当前窗口表格，但不展示强趋势判断。

## 边界情况

- `REPORT_PERSPECTIVE=window` 时，趋势页仍应通过历史查询函数读取历史库，而不是退回本次窗口。
- 回放旧报告时，趋势数据必须以该报告 `data_until` 为截止，不能读到未来才入库的数据。
- 后采集到的历史评论不能进入旧报告回放；例如 2026-05-01 才入库、发布时间为 2026-04-01 的评论，不能出现在 2026-04-30 的历史趋势里。
- 模拟 30 天报告时，近30天可展示当前窗口趋势；前30天不足时，对比区显示“暂无足够历史对比”。
- 近12个月可以展示已有月份趋势；前12个月不足时不展示同比变化。
- 评论日期为相对时间时，以入库时解析出的 `date_published_parsed` 为准，避免每次生成报告重新漂移。

## 验收标准

- 变化趋势页顶部有时间和维度切换
- 每个维度最多一张主图、三个关键指标、一张明细表
- 页面不出现“快照”“声浪”“声量”“上期”
- 对比文案必须写明具体窗口
- 趋势数据截止到报告 `data_until`，旧报告回放不读未来数据
- 近30天模拟报告中能看到口碑趋势、问题趋势、产品趋势、竞品对比四个维度
- 历史不足时不展示伪精确变化
